go-order-management-system

一个带 JWT 鉴权、用户订单隔离、商品库存、订单状态机、事务一致性、幂等下单、Redis 缓存和 Docker 编排能力的轻量级商品订单管理系统。

查看 GitHub 仓库 查看表设计 查看业务规则

项目定位

这个项目不是简单 CRUD，而是面向 Go 后端求职场景，用订单、库存、用户鉴权和缓存这些常见业务模块展示真实后端工程能力。

  • handler / service / dao / model 分层清晰
  • 使用事务保证订单创建、库存扣减和库存流水一致性
  • 使用 idempotency_key 处理重复下单问题
  • 通过 JWT 和 user_id 查询条件实现订单归属隔离

技术栈

Go Gin GORM MySQL Redis JWT Goose Docker GitHub Actions

商品模块

  • 商品创建、查询、上架、下架
  • 商品价格使用分为单位
  • 商品详情 Redis cache-aside 缓存

库存模块

  • 库存初始化与增加
  • 库存流水记录
  • 下单扣减与取消回滚

订单模块

  • 幂等创建订单
  • 支付、完成、取消状态流转
  • 用户级订单数据隔离

本地运行方式

项目后端依赖 MySQL 和 Redis，推荐通过 Docker Compose 本地启动。

docker compose up --build

服务默认暴露端口：8082。健康检查接口：/ping 和 /readyz。

部署说明

GitHub Pages 只适合部署当前这个静态展示页，不能运行 Go/Gin 后端、MySQL 或 Redis。真正的后端部署应使用云服务器、容器平台、Render、Railway、Fly.io、Koyeb 或其他支持常驻服务和数据库的环境。

面试讲解重点

  • 为什么订单创建、库存扣减、订单项写入和库存流水要放在同一个事务内。
  • 为什么要使用用户级 idempotency_key，以及如何处理相同 key 不同请求的冲突。
  • 订单状态机如何防止非法状态流转。
  • Redis 缓存不可用时为什么应该降级而不是阻断主流程。
  • Docker Compose、CI、迁移工具和健康检查如何体现工程化能力。
