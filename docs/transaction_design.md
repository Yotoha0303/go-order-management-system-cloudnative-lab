# 事务设计说明

## 创建订单事务

以下操作在同一个 MySQL 事务中完成：

1. 抢占用户级幂等记录并校验请求摘要；
2. 创建订单主表；
3. 按商品读取并锁定库存；
4. 条件扣减库存；
5. 写入订单明细和库存流水；
6. 写入订单超时 Outbox；
7. 更新订单总金额并完成幂等记录。

任一步失败时整体回滚，避免出现订单存在但库存、流水或超时事件缺失。

## 取消订单事务

- 使用旧状态条件把订单从 `pending` 更新为 `cancelled`。
- 逐项锁定库存、回补数量并写入回滚流水。
- 任一商品回补失败时，订单状态和已完成的回补全部回滚。

## 事务外操作

- RabbitMQ 发布由 Outbox Worker 在事务提交后执行，Publisher Confirm 成功后更新发布状态。
- Redis 只缓存商品详情，不参与库存正确性事务。
- HTTP、日志和外部消息调用不放在数据库事务持锁区间内。

详细时序见 [order_flow.md](order_flow.md)，幂等规则见 [idempotency.md](idempotency.md)。
