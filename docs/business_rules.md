# 业务规则说明

## 1. 商品模块

### 创建商品

规则：

1. 商品名称不能为空
2. 商品价格 price_fen 必须大于 0
3. 商品创建后默认下架，status = 2

### 查询商品列表

规则：

1. 不传 status 时默认查询下架商品，保持原接口行为
2. status = 1 查询上架商品，status = 2 查询下架商品，status = all 查询全部商品
3. page 默认 1，page_size 默认 20 且最大 100；仅在传入分页参数时返回分页对象
4. 下架商品用于库存初始化和上架前准备，上架商品用于订单创建

### 商品上架

规则：

1. 商品必须存在
2. 商品状态修改为 status = 1
3. 已上架商品重复上架直接成功

### 商品下架

规则：

1. 商品必须存在
2. 商品状态修改为 status = 2
3. 已下架商品重复下架直接成功

## 2. 库存模块

### 初始化库存

规则：

1. product_id 必须存在
2. stock_quantity 必须大于等于 0
3. 一个商品只能初始化一次库存
4. 初始化库存后必须写入 stock_logs
5. 库存表使用 product_inventories

### 增加库存

规则：

1. quantity 必须大于 0
2. 商品库存记录必须已存在
3. 增加库存前记录 before_quantity
4. 增加库存后记录 after_quantity
5. 每次增加库存都要写 stock_logs
6. biz_type = 2 表示手动入库

## 3. 库存流水模块

规则：

1. product_id 可选
2. 不传 product_id 查询全部库存流水
3. 传 product_id 查询指定商品库存流水
4. 按 created_at desc 排序
5. biz_type = 1 表示初始化库存
6. biz_type = 2 表示手动入库
7. biz_type = 3 表示订单扣减库存
8. biz_type = 4 表示取消订单回滚库存

## 4. 用户与鉴权模块

规则：

1. 用户名去除首尾空格后长度为 3 到 64 个字符
2. 密码长度为 6 到 72 个字符，数据库只保存 bcrypt 哈希
3. JWT Secret 至少 32 个字符，令牌必须包含有效 user_id、username、issuer、issued_at 和 expires_at
4. 用户资料与订单接口从 JWT 身份读取 user_id，不接受客户端自行提交用户 ID

## 5. 订单模块

### 创建订单

规则：

1. items 不能为空
2. product_id 必须大于 0
3. quantity 必须大于 0
4. 商品必须存在
5. 商品必须已上架
6. 商品库存必须存在
7. 库存必须充足
8. idempotency_key 必填且长度不能超过 128
9. 订单必须归属于当前 JWT 用户
10. 同一用户相同 idempotency_key 和相同请求内容返回原订单，不重复扣减库存
11. 同一用户相同 idempotency_key 和不同请求内容返回冲突；不同用户可使用相同 Key
12. 幂等记录、创建订单、扣减库存、创建订单项、写库存流水必须在同一个事务内完成
13. 查询、支付、完成、取消都必须同时匹配 order_id 和当前 user_id

### 支付订单

允许：

- 待支付 -> 已支付

禁止：

- 已支付订单重复支付
- 已取消订单支付
- 已完成订单支付

### 完成订单

允许：

- 已支付 -> 已完成

禁止：

- 未支付订单完成
- 已取消订单完成
- 已完成订单重复完成

### 取消订单

允许：

- 待支付 -> 已取消

禁止：

- 已支付订单取消
- 已完成订单取消

取消订单时需要回滚库存，并写入 biz_type = 4 的库存流水。

### 取消订单幂等规则

已取消订单再次调用取消接口时，直接返回成功，不重复回滚库存。

原因：

1. 避免前端重复点击取消按钮导致错误响应
2. 避免网络重试导致重复业务处理
3. 保证库存不会被重复回滚

## 6. Redis 缓存

### 商品模块

规则：
1. 查询商品时，设置商品缓存
2. 上架或下架商品时，商品缓存删除
3. Redis 不启用或者启动异常，不会影响商品模块的正常查询
